## 1. 需求

### 背景
当前代码库是基于 `Lampese/codex-switcher` 持续改出来的 Windows 可用版本，但原公开仓库已经不再适合 Windows 场景，后续也没有必要继续围绕原仓库做合并或提交。需要把项目整理成独立仓库形态，完成统一重命名，并明确说明这是基于原项目的扩展与修正版本。

### 目标
- 将项目对外名称统一重命名为 `codex-account-switcher`。
- 在 README 和应用元数据中明确说明：这是基于原项目的独立扩展与修正，重点补强 Windows 支持、多账号切换和稳定性。
- 清理仍指向原仓库品牌或发布地址的关键元数据，避免独立发布后继续误导到上游仓库。

### 约束条件
```yaml
时间约束: 本轮内完成独立仓库化重命名并验证可构建
性能约束: 不涉及运行时性能优化，不引入额外后台负担
兼容性约束: 尽量保留现有本地账号数据兼容性，不让已保存的账号配置失效
业务约束: 不再面向原仓库合并；说明中需明确“基于原项目的扩展和修正”
```

### 验收标准
- [ ] `package.json`、`Cargo.toml`、`tauri.conf.json`、页面标题和 README 的主要对外名称统一切换到 `codex-account-switcher` / `Codex Account Switcher`。
- [ ] README 明确写明该项目基于 `Lampese/codex-switcher` 扩展，并说明 Windows 支持与修正内容。
- [ ] 原仓库的 updater/release 指向不再保留为默认发布地址，避免新仓库继续指向上游 releases。
- [ ] `bun run build` 与 `bun run tauri:build` 通过。

## 2. 方案

### 技术方案
以“对外品牌统一、内部数据兼容优先”为原则做独立仓库化调整：前端包名、Tauri 产品名、窗口标题、浏览器标题、README 标题与介绍、备份文件名、版本脚本匹配目标等统一切到 `codex-account-switcher`；README 增加来源说明与独立定位说明。对于本地账号数据目录和现有配置存储路径，先保持 `~/.codex-switcher` 兼容，避免现有用户数据迁移成本。对 updater 端点移除原仓库发布地址，避免误连上游 releases。

### 影响范围
```yaml
涉及模块:
  - package-tooling: npm/bun 包名、版本脚本、构建文档
  - tauri-commands: 产品元数据、应用标题、发布配置
  - frontend-app: 页面标题、应用显示名称、备份文件名
  - knowledge-base: 模块文档与变更记录同步
预计变更文件: 8-12
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 改动内部存储目录导致现有账号数据丢失 | 中 | 本轮不改 `~/.codex-switcher` 路径，先保持兼容 |
| updater 仍指向原仓库导致新仓库发布混乱 | 中 | 移除原仓库发布端点，README 补充独立发布说明 |
| 品牌名替换不完整导致新旧名称混用 | 低 | 全局搜索主要品牌词，覆盖关键元数据和用户可见文案 |

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### rebrand-independent-repo#D001: 对外独立改名，但保留本地数据目录兼容
**日期**: 2026-04-14
**状态**: ✅采纳
**背景**: 用户要把当前版本作为新仓库独立发布，但当前项目已经积累了本地账号数据和 Windows 适配修复。若同时重命名对外品牌和本地数据目录，会把“仓库独立”与“存储迁移”绑在一起，增加不必要风险。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 对外名称和内部存储目录一起改 | 品牌最统一 | 需要处理已有 `~/.codex-switcher` 数据迁移，容易引入兼容问题 |
| B: 对外品牌统一改名，内部存储目录暂时保留 | 可立即独立发布，且不破坏现有用户数据 | 内部仍会保留少量历史命名痕迹 |
**决策**: 选择方案 B
**理由**: 当前目标是独立仓库化和对外重命名，不是做一次高风险的数据迁移。先保证现有用户数据和 Windows 使用体验稳定，后续若真要迁移目录，可以单独做带迁移逻辑的版本。
**影响**: 影响 `package.json`、Tauri 元数据、README、前端显示文案、版本脚本；`auth-storage` 路径本轮不改
